# Cursor Workflow - Xcode & Git Integration

## Automatic Process

Every time code changes are made, ensure:

1. ✅ **All new files are added to Xcode project** (`project.pbxproj`)
2. ✅ **All changes are committed to git** with descriptive messages
3. ✅ **Verify no uncommitted changes remain**

## Checklist Before Completing Tasks

- [ ] New Swift files added to `project.pbxproj`:
  - PBXBuildFile section
  - PBXFileReference section  
  - PBXGroup (appropriate group)
  - PBXSourcesBuildPhase
  
- [ ] All changes committed to git
- [ ] Git status is clean (no uncommitted changes)
- [ ] No linter errors

## Quick Verification Commands

```bash
# Check git status
git status --short

# Verify files in Xcode project
grep -E "YourNewFile.swift" ios/WineLensApp/.../project.pbxproj

# Check for uncommitted changes
git diff --name-only
```

## Standard Workflow

1. Make code changes
2. Add files to Xcode project if new
3. Run linter check
4. Stage all changes: `git add -A`
5. Commit with descriptive message
6. Verify: `git status`
